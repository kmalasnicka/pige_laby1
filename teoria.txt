SendMessage - działa od razu i czeka na wynik, zrob to teraz
PostMessage - wrzuca wiadomość do kolejki wiec zostaje obsluzona później, zrob później przez queue

np.: STATIC z napisem i po klikniecu guzika chcemy od razu zmienić tekts
SendMessageW(hLabel, WM_SETTEXT, 0, (LPARAM)L"Nowy tekst"); //SETTEXT ma być wykonane od razu

np.: po kliknieciu klawisza dodaj zdarzerzenie do obsługi później
PostMessageW(m_main, WM_APP + 1, 0, 0); // wysylanie sama sobie custom message
w window_proc:
case WM_APP + 1:
    // zrób coś później
    return 0;

GetMessage - czeka na wiadomość i blokuje program
PeekMessage - nie czeka, sprawdza czy cos jest w kolejce, nie blokuje

np.: kulka ma się poruszać caly czas ale program nadal ma reagować na klawiaturę i zamykanie okn a
MSG msg{};
bool done = false;

while (!done)
{
    if (PeekMessageW(&msg, nullptr, 0, 0, PM_REMOVE))
    {
        if (msg.message == WM_QUIT)
            done = true;
        else
        {
            TranslateMessage(&msg);
            DispatchMessageW(&msg);
        }
    }
    else
    {
        // tu animacja bez czekania
        // update pozycji
    }
}

Focus - kto dostaje klawiaturę

Z-order - kolejność okien które jest nad którym
SetWindowPos - zmienia pozycję, rozmiar i też Z-order
SetWindowPos(hLabel, HWND_TOP, x, y, w, h, SWP_NOMOVE | SWP_NOSIZE); //okno które zmieniamy, gdzie w kolejności okien przenieś to okno na sama gore z-order, nowa pozycja x, nowe y, szerokość, wysokość, co dokładnie zmieniamy
SWP_NOMOVE - nie zmieniaj pozycji
SWP_NOSIZE - nie zmieniaj rozmiaru
